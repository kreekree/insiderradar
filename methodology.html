Data & Methodology

Exactly where the data comes from, how we parse it, and what every calculation means. No black boxes. Last updated: April 2026.

Data Sources

InsiderRadar uses only official, publicly available data from the US Securities and Exchange Commission. No third-party data vendors are used at any stage.

SEC Company Tickers JSON

https://www.sec.gov/files/company_tickers.json

The SEC maintains a JSON file mapping every registered public company's stock ticker to its CIK (Central Index Key) number. We cache this file in memory for 24 hours and use it to resolve ticker symbols entered by users into SEC-compatible CIK numbers.

SEC EDGAR Submissions API

https://data.sec.gov/submissions/CIK{cik}.json

For each company, we fetch a structured JSON document listing all SEC filings in reverse chronological order. This document includes the form type, filing date, accession number, and primary document name for each filing. We filter this list to identify the most recent Form 4 filings (up to 15).

SEC EDGAR Form 4 XML Documents

https://www.sec.gov/Archives/edgar/data/{cik}/{accession}/{document}.xml

Each Form 4 filing is stored as an XML document in the SEC EDGAR Archives. We fetch and parse these XML files to extract transaction-level data including the insider's name, title, transaction date, transaction code, number of shares, price per share, and post-transaction share count.

SEC EDGAR Full-Text Search (EFTS)

https://efts.sec.gov/LATEST/search-index

For the Unusual Activity page and Fund Tracker, we use the SEC's full-text search API to discover recent Form 4 and 13F-HR filings. This endpoint returns filing metadata including entity name, filing date, and accession number, which we then use to retrieve the underlying XML documents.

How We Parse Form 4 XML

SEC Form 4 filings are structured XML documents following a consistent schema. We extract the following fields using tag-pattern matching:

  • rptOwnerName — The insider's full legal name
  • isOfficer / officerTitle — Whether the person is an officer and their title (CEO, CFO, etc.)
  • isDirector — Whether the person serves on the board of directors
  • issuerTradingSymbol — The stock ticker symbol
  • nonDerivativeTransaction blocks — Each block represents one open-market transaction
  • transactionCode — P (purchase), S (sale), A (award), M (option exercise), F (tax withholding), etc.
  • transactionShares/value — Number of shares traded
  • transactionPricePerShare/value — Price paid or received per share
  • transactionDate/value — Date of the transaction
  • sharesOwnedFollowingTransaction/value — Total shares held after the transaction

We only parse <nonDerivativeTransaction> elements, which represent direct stock transactions. Derivative transactions (options, warrants) are excluded from the main table but may be added in a future update.

How "Unusual Activity" Is Calculated

A transaction is flagged as unusual when both of the following conditions are met:

  1. The transaction code is P (open-market purchase) — not an award, option exercise, or other non-discretionary transaction
  2. The total transaction value (shares × price) exceeds $500,000 in a single trade

The $500,000 threshold was selected to filter out small routine purchases while surfacing meaningful capital commitments. Future versions may add a dynamic threshold based on each insider's historical average purchase size.

How We Parse 13F-HR Holdings

13F filings consist of two documents: a cover page (XML) and an InfoTable (XML) containing the actual holdings. We parse the InfoTable to extract:

  • nameOfIssuer — Company name
  • titleOfClass — Security class (usually "COM" for common stock)
  • cusip — 9-character security identifier
  • value — Fair market value in thousands of dollars (we multiply by 1,000)
  • sshPrnamtAmt — Number of shares held
  • investmentDiscretion — SOLE or SHARED
  • putCall — Options flag (Put or Call, if applicable)

Holdings are sorted by value descending and limited to the top 50 positions for display. The portfolio weight percentage shown is calculated as: (position value / total portfolio value) × 100.

Caching & Rate Limits

The SEC requests that all automated access include a descriptive User-Agent header and does not impose hard rate limits for reasonable use. All InsiderRadar API responses are cached at the Vercel edge layer for 1 hour (s-maxage=3600). The company tickers JSON is cached in-memory for 24 hours. This means:

  • Results may be up to 1 hour behind the live SEC EDGAR database
  • Repeated searches for the same ticker within 1 hour will return cached results instantly
  • For the most current data, search directly on SEC EDGAR
